Integrity methodology

How the 99%+ number is actually calculated

Not a vibe. Not a placeholder. Here's the exact formula Blackloom runs on every pipeline execution, and why it's built this way.

Four dimensions, not one score

Real data-quality platforms — Great Expectations, Monte Carlo — score pipelines on independent axes because a single "integrity %" hides which kind of problem you have. Blackloom does the same: four scores, each computed separately, then combined with weights that reflect what actually breaks a pipeline in production.

Completeness
30%
Fraction of raw rows that still have every required field after validation.
Uniqueness
20%
Fraction of rows that are not duplicates, by transaction hash.
Validity
35%
Fraction passing business rules — value ≥ 0, gas price > 0, block number > 0.
Timeliness
15%
How fresh the newest record is. Decays linearly to 0% at 30 days old.
The exact formula

overall_score = Σ (dimension_score × weight)

completeness = valid_rows / raw_rows × 100 uniqueness = (1 − duplicate_rows / total_rows) × 100 validity = rows_passing_business_rules / total_rows × 100 timeliness = max(0, 100 − (hours_since_newest_record / 720) × 100) overall_score = completeness × 0.30 + uniqueness × 0.20 + validity × 0.35 + timeliness × 0.15

Validity carries the heaviest weight (35%) because bad values — negative amounts, impossible gas prices — are the failure mode most likely to silently corrupt downstream analysis. Completeness is second because missing fields can't be recovered after the fact. This is exactly the logic implemented in src/quality.py — nothing here is simplified for the page.

Worked example

A real run, from actual pipeline output

raw_rows = 1024, valid_rows = 1023 → completeness = 99.90% duplicates = 20 of 1023 rows → uniqueness = 98.04% all values ≥ 0, gas > 0, block > 0 → validity = 100.00% newest record 2 hours old → timeliness = 99.72% overall = 99.90(.30) + 98.04(.20) + 100.00(.35) + 99.72(.15) = 29.97 + 19.61 + 35.00 + 14.96 = 99.54%

That's a real number from a real run, not a rounded-up marketing figure — see it live on the dashboard, recalculated every scheduled run.
